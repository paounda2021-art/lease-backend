# ระบบติดตามหนี้เช่า — Backend + Database (Lease Receivables Tracker)

ระบบหลังบ้านจริงสำหรับติดตามลูกหนี้ค่าเช่า เก็บข้อมูล **ถาวร** ในฐานข้อมูล SQLite
รองรับพอร์ต 500+ สัญญา พร้อม REST API และหน้าเว็บที่ต่อ API จริง

สแตก: **Node.js 22 + Express + node:sqlite** (ฐานข้อมูลในตัว Node ไม่ต้องคอมไพล์ ไม่ต้องติดตั้ง DB แยก)

---

## ความต้องการ

- Node.js เวอร์ชัน **22.5.0 ขึ้นไป** (ใช้โมดูล `node:sqlite` ในตัว)
  ตรวจเวอร์ชัน: `node -v`

## การติดตั้งและรัน (3 ขั้น)

```bash
npm install          # ติดตั้ง express
npm run seed         # สร้างข้อมูลตัวอย่าง (6 สัญญา ~213 ใบแจ้งหนี้) ลงไฟล์ lease.db
npm start            # เปิดเซิร์ฟเวอร์
```

เปิดเบราว์เซอร์ที่ **http://localhost:3000**

> ข้าม `npm run seed` ได้ถ้าต้องการเริ่มจากฐานข้อมูลว่าง (ระบบสร้างตารางให้อัตโนมัติเมื่อรันครั้งแรก)

ตั้งค่าเพิ่มเติมผ่าน environment variable:
- `PORT` — พอร์ตของเซิร์ฟเวอร์ (ค่าเริ่มต้น 3000)
- `DB_PATH` — ตำแหน่งไฟล์ฐานข้อมูล (ค่าเริ่มต้น `./lease.db`)

## โครงสร้างไฟล์

| ไฟล์ | หน้าที่ |
|---|---|
| `schema.sql` | โครงสร้างฐานข้อมูล (7 ตาราง) |
| `db.js` | เชื่อมต่อ SQLite + สร้างตาราง + audit helper |
| `aging.js` | ตรรกะจัดชั้นอายุหนี้ / provision / จัดชั้น ก.186 |
| `server.js` | REST API (Express) |
| `seed.js` | สร้างข้อมูลตัวอย่าง |
| `public/index.html` | หน้าเว็บ (ต่อ API จริง) |

## ตารางในฐานข้อมูล

`customers` · `contracts` · `invoices` · `payments` · `dunning_log` · `provision_rates` · `audit_log`

ทุกการกระทำสำคัญ (รับชำระ / ตัดหนี้สูญ / แก้อัตราสำรอง) ถูกบันทึกใน `audit_log` เพื่อการตรวจสอบ

## REST API หลัก

| Method | Endpoint | หน้าที่ |
|---|---|---|
| GET | `/api/dashboard?asof=YYYY-MM-DD` | KPI สรุป (AR, overdue, provision, ฟ้อง, ตัดสูญ) |
| GET | `/api/contracts` | รายการสัญญา + ยอดค้างต่อราย |
| POST/PUT/DELETE | `/api/contracts[/:id]` | สร้าง/แก้ไข/ลบสัญญา |
| GET | `/api/invoices?asof=&filter=` | ใบแจ้งหนี้ + aging + การกระทำแนะนำ (filter: open/overdue/litig/wo) |
| POST | `/api/invoices/generate` | สร้างใบแจ้งหนี้งวดถัดไป (`{contract_id:"all"}` = ทุกสัญญา) |
| POST | `/api/invoices/:id/payment` | รับชำระ (`{amount, wht, method}`) |
| POST | `/api/invoices/:id/use-deposit` | หักจากเงินประกัน |
| POST | `/api/invoices/:id/litigation` | ธงส่งฟ้อง |
| POST | `/api/invoices/:id/writeoff` | ตัดหนี้สูญ (**บังคับ** ส่ง `{reason}` ตาม ก.186) |
| GET | `/api/aging?asof=` | สรุป aging ตาม bucket + รายสัญญา |
| GET/PUT | `/api/provision[/rates]` | provision matrix + แก้อัตราสำรอง |
| POST | `/api/dunning` | บันทึกการทวง (หลักฐาน ก.186) |
| POST | `/api/jobs/daily` | งานรายวัน (จำลอง cron): สรุปการทวงที่ต้องทำ |
| GET | `/api/audit` | ประวัติการแก้ไข |

## สิ่งที่ระบบทำให้ถูกหลักการ

- **จัดชั้นอายุหนี้อัตโนมัติ** จากวันครบกำหนดของแต่ละใบแจ้งหนี้ (7 ช่วง)
- **ตั้งสำรอง (provision)** ตามตารางอัตราที่แก้ไขได้ เก็บอัตราลงฐานข้อมูล
- **จัดชั้นตามกฎกระทรวง 186** ต่อยอดหนี้รวมรายลูกหนี้ (>2ล้าน / 2แสน–2ล้าน / ≤2แสน)
- **บังคับให้ระบุเหตุผล/หลักฐานก่อนตัดหนี้สูญ** — ตัดโดยไม่มีเหตุผลจะถูกปฏิเสธ
- **Audit trail** ทุกการกระทำ เพื่อการตรวจสอบและป้องกันทุจริต

## นำไปใช้จริงต่อ (production)

1. เพิ่ม **authentication** และสิทธิ์ผู้ใช้ (คนวางบิล/รับเงิน/อนุมัติตัดหนี้ ต้องแยกกัน)
2. ตั้ง **cron** ยิง `POST /api/jobs/daily` ทุกเช้า + ต่อระบบส่งอีเมล/SMS ทวงอัตโนมัติ
3. เชื่อม **โปรแกรมบัญชี** (ส่ง journal entry อัตโนมัติ) — ดูผังบัญชี/รายการลงบัญชีในเอกสาร SOP
4. ย้ายจาก SQLite เป็น **PostgreSQL** เมื่อผู้ใช้พร้อมกันหลายคน (โครง schema ใช้ต่อได้เลย)
5. สำรองข้อมูล (backup ไฟล์ `lease.db` เป็นประจำ)
